3.1 Introduction to Unsupervised Learning — K-Means, DBSCAN, Apriori, FP-Growth

What is Unsupervised Learning?

Unsupervised Learning is a type of machine learning where the model works without labelled data. It learns patterns on its own by grouping similar data points or finding hidden structures without any human intervention.

  • Used for tasks like clustering, dimensionality reduction, and association rule learning
  • Helps identify hidden patterns in data
  • Useful for grouping, compression, and anomaly detection

How Unsupervised Learning Works

Collect Unlabeled Data — gather a dataset without predefined labels or categories
Select an Algorithm — choose clustering (K-Means), association rule learning (Apriori), or dimensionality reduction (PCA) based on the goal
Train on Raw Data — feed the unlabeled dataset; the algorithm finds similarities, relationships, or hidden structures
Group or Transform Data — algorithm organizes data into clusters, rules, or lower-dimensional forms without human input
Interpret and Use Results — analyze discovered groups or rules for visualization, anomaly detection, or further modelling

Three Types of Unsupervised Algorithms

1. Clustering

Groups unlabeled data into clusters based on similarity. Used for customer segmentation, anomaly detection, data organization.

2. Association Rule Learning

Discovers "if-then" relationships between variables in large datasets. Used in market basket analysis, recommendation systems.

3. Dimensionality Reduction

Decreases the number of features while retaining information. Reduces noise and computational cost. Algorithms: PCA, LDA, NMF, LLE, Isomap.

K-Means Clustering

K-Means groups similar data points into clusters without needing labeled data. It computes centroids and iterates until the optimal centroid is found. It assumes the number of clusters K is already known.

Data points are assigned to clusters such that the sum of squared distances between points and their centroid is minimized. Less variation within clusters means more similar data points.

Steps of K-Means Algorithm

Specify the number of clusters K
Randomly select K data points and assign each to a cluster
Compute the cluster centroids
Iterate: compute sum of squared distances → reassign each point to the nearest centroid → recompute centroids
Stop when cluster assignments no longer change

Applications of K-Means

Data Segmentation

Group customers based on purchasing patterns or website interaction.

Image Compression

Group similar pixels into clusters to reduce image complexity and storage.

Anomaly Detection

Identify data points that do not belong to any of the clusters.

Document Clustering

Group similar documents in NLP for recommendation systems or news categorization.

DBSCAN

DBSCAN (Density-Based Spatial Clustering of Applications with Noise) groups data points that are closely packed and marks outliers as noise based on their density in the feature space. It identifies clusters as dense regions separated by areas of lower density.

Key Parameters

Eps (ε — Epsilon)

The radius of the neighborhood around a data point. Points within ε distance are considered neighbors. If ε is too small → most points become noise. If ε is too large → clusters may merge.

MinPts

Minimum number of points required within the ε radius to form a dense region. General rule: MinPts ≥ D + 1, where D is the number of dimensions.

Working of DBSCAN

Randomly select an unvisited data point
If it has ≥ MinPts neighbors within distance ε, create a new cluster and add the point and its neighbors
If it does not meet the MinPts threshold, mark it as noise
Repeat until all data points have been visited

Advantages of DBSCAN over K-Means

  • Handles clusters of arbitrary shape (not just spherical)
  • Does not require knowing K in advance
  • Can detect and label outliers/noise
  • Relatively insensitive to initial parameter choice
  • Scalable to large datasets — only computes pairwise distances between neighboring points

Association Mining with Apriori

Apriori is the most widely used method for discovering association rules. It identifies frequent itemsets — groups of items that appear together in transactions — and derives "if-then" rules describing their relationships.

Apriori Property: If an itemset is frequent, all its subsets must also be frequent. If an itemset is infrequent, all its supersets are also infrequent. This prunes the search space and improves efficiency.

Key Metrics

Support

How often an item or combination appears in all transactions. Higher support = more significant presence.
e.g., Bread appears in 20% of transactions.

Confidence

Likelihood that item Y is purchased when item X is purchased. Measures strength of association.
e.g., If bread → butter 75% of the time.

Lift

How much more likely two items are bought together vs. independently. Lift > 1 = strong positive association.

Steps of Apriori Algorithm

Set parameters — define minimum support threshold and minimum confidence threshold
Find frequent 1-itemsets — count occurrences of each single item; keep those meeting the support threshold
Generate candidate 2-itemsets — combine frequent 1-itemsets into pairs; calculate their support; prune infrequent ones
Extend to 3-itemsets and beyond — repeat until no more frequent itemsets can be found
Generate association rules — from frequent itemsets, generate rules and calculate confidence; keep rules meeting the confidence threshold

Applications of Apriori

  • E-commerce: Recommend products frequently bought together (laptop + laptop bag)
  • Food Delivery: Identify popular combos (burger + fries) for combo deals
  • Streaming Services: Recommend related content based on co-viewing patterns
  • Financial Services: Analyze spending habits to suggest personalized offers
  • Health & Fitness: Suggest workout plans based on past activity patterns

FP-Growth Algorithm

FP-Growth (Frequent Pattern Growth) efficiently mines frequent itemsets from large transactional datasets. Unlike Apriori, which suffers from high computational cost due to candidate generation and multiple database scans, FP-Growth avoids these inefficiencies by compressing data into an FP-Tree and extracting patterns directly from it.

How FP-Growth Works

Compute item frequencies — scan dataset once to count occurrence of each item
Order items by frequency — sort every transaction in descending order by item frequency; allows FP-tree compression
Construct the FP-Tree — insert sorted transactions one by one, merging shared prefixes
Determine conditional pattern bases — for each item, collect all prefix paths leading to it
Build conditional FP-Trees — construct smaller FP-trees from each conditional pattern base
Extract frequent itemsets — mine all frequent itemsets from the conditional trees

Why FP-Growth is Efficient

  • Scans the database only twice
  • Avoids generating all candidate item combinations
  • Stores data in a compact tree, reducing redundancy
  • Uses conditional trees to mine deeper patterns efficiently
Apriori vs FP-Growth: Apriori performs multiple database scans and generates candidate itemsets at every level — slow for large datasets. FP-Growth compresses everything into an FP-Tree with just 2 scans, making it significantly faster and more memory-efficient.
3.2 Anomaly Detection — Isolation Forest, Autoencoders

What is Anomaly Detection?

Anomaly detection (also known as outlier detection) is the process of identifying data points, events, or patterns that deviate significantly from expected behavior in a dataset. These anomalies often indicate critical issues such as fraud, system failures, or security breaches.

  • Used across finance, healthcare, manufacturing, and cybersecurity
  • Closely coupled with time series analysis — anomalies are deviations from past norms
  • Early detection can prevent cascading damage (e.g., power grid failures, financial fraud)

Applications of Anomaly Detection

Fraud Detection

Identifying unusual financial transactions in banking and insurance.

Cybersecurity

Detecting irregular network traffic or unauthorized access attempts.

Healthcare

Monitoring patient vitals for abnormal patterns indicating health issues.

Manufacturing

Identifying defects in production lines or predicting equipment failures.

IT Operations

Monitoring server logs for unusual patterns to prevent system failures.

Importance in Cybersecurity

Anomaly detection identifies deviations in user activity, network behavior, or system events that indicate potential threats. Types of anomalies include:

  • Unusual login patterns: Login at odd hours, from foreign IPs, or multiple geographic locations — indicates credential compromise
  • Privilege escalation: Standard user executing admin commands or accessing restricted resources
  • Lateral movement: Account or process accessing systems it typically never interacts with
  • Data exfiltration: Sudden surge in outbound traffic involving sensitive data or unrecognized destinations
  • Unusual process behavior: Processes spawning unexpectedly, executing from non-standard directories, or making unauthorized registry changes
  • Application misuse: Users performing actions outside their normal usage pattern — e.g., downloading reports in bulk
  • Unexpected service activity: New ports being opened, services restarting frequently, or protocols used in unexpected ways
  • Time-based anomalies: Deviations in CPU usage, memory spikes, or authentication attempt rates over time

Isolation Forest

Isolation Forest is highly effective in identifying outliers in large datasets. It isolates anomalies based on how easily they can be separated from the rest of the data. Built on decision trees (like Random Forests), it is an unsupervised model with no pre-defined labels.

Key insight: anomalies are few and different — they are easier to isolate and require fewer random cuts in a tree. Normal points require deeper trees to isolate.

How Isolation Forest Works

A random sub-sample of data is selected and assigned to a binary tree (Isolation Tree / iTree)
Branching starts by selecting a random feature, then a random threshold (between the feature's min and max values)
Data points with value less than the threshold go left; otherwise right — creating a binary split
Process repeats recursively until each point is isolated or max depth is reached
This is repeated to build an ensemble of iTrees (the Isolation Forest)
During scoring, each data point is traversed through all iTrees. An anomaly score is assigned based on depth — shallow depth = anomaly (score = −1), deep = normal (score = 1)
Samples that travel deeper into the tree are less likely to be anomalies (needed more cuts). Samples in shorter branches are anomalies (easier to separate). Anomaly score of −1 = anomaly; +1 = normal.

Use in Cybersecurity

Isolation Forest is commonly used for detecting rare attack patterns such as privilege misuse or unusual file access.

Autoencoders for Anomaly Detection

Autoencoders are neural networks designed to learn efficient representations of data through an encode-decode process. The network consists of:

  • Encoder: Compresses input data into a lower-dimensional latent representation by progressively reducing neuron counts per layer
  • Decoder: Reconstructs the original data from the compressed representation through layers that progressively increase in size

How Autoencoders Detect Anomalies

Autoencoders trained on normal data become highly efficient at reconstructing similar normal patterns but struggle to reconstruct anomalous patterns they haven't seen during training. This difference in reconstruction quality — measured as reconstruction error — is the basis for anomaly detection. High reconstruction error = likely anomaly.

Why Autoencoders are Useful

Unsupervised Learning Capability

Don't require labeled anomalous data during training — practical since anomalies are rare and hard to collect. They learn normal data structure and detect deviations.

High-Dimensional Data Handling

Traditional methods struggle with the curse of dimensionality. Autoencoders naturally handle high-dimensional inputs by learning meaningful lower-dimensional representations.

Non-Linear Pattern Recognition

Unlike PCA-based detection, autoencoders capture complex non-linear relationships through deep architecture, detecting subtle anomalies in complex datasets.

Adaptive Threshold Learning

No manual threshold setting required — systems learn appropriate decision boundaries from the distribution of reconstruction errors observed during training.

3.3 Clustering Methods for Cybersecurity Threat Detection

Overview

Clustering methods are instrumental in identifying cybersecurity threats by grouping similar data points to uncover patterns and anomalies. These methods enhance efficiency and precision of threat detection, enabling faster incident response and reduced false positives.

Clustering Algorithms and Their Cyber Applications

K-Means

Partition-based clustering dividing data into K clusters based on similarity to centroids. In cybersecurity, used to group network traffic and identify deviations from normal patterns. Data points far from any cluster centroid may indicate suspicious activity.

DBSCAN

Density-based clustering, very effective for anomaly detection. Forms clusters in dense regions and marks low-density points as noise. Widely used to detect DDoS attacks — sudden traffic spikes create abnormal patterns that stand out from normal behavior.

Hierarchical Clustering

Creates a tree-like structure (dendrogram) representing relationships between data points. In cybersecurity, helps analyze similarities between attack patterns and group different types of malware or intrusion activities based on behavior.

Gaussian Mixture Models (GMM)

Probabilistic clustering method assuming data is generated from multiple distributions. Useful for modeling complex behaviors in network traffic and detecting subtle anomalies not easily identified by simple clustering methods.

Applications of Clustering in Cybersecurity

1. Intrusion Detection Systems (IDS)

Clustering detects unauthorized access by identifying unusual patterns in network traffic. Activities that do not belong to normal clusters are flagged as potential intrusions.

2. Malware Detection

Files and programs are grouped based on behavior or structure. Similar malware samples form clusters; unknown or new malware appears as separate clusters, enabling early detection.

3. Network Anomaly Detection

Clustering identifies unusual network behavior such as sudden traffic increases, unexpected communication patterns, or suspicious IP activity — indicators of cyber attacks.

4. User Behavior Analysis

User activities (login times, access patterns, system usage) are clustered. Deviations from normal user behavior clusters indicate insider threats or compromised accounts.

5. Phishing and Fraud Detection

Clustering groups similar emails or URLs. Suspicious or malicious ones appear as anomalies and can be flagged for further investigation.

6. Endpoint Configuration Monitoring

Periodic snapshots of endpoint configurations are clustered to identify outlier endpoints with irregular configurations. PCA is used to improve efficiency; original feature values are used for explainability.

Key Takeaway: Clustering doesn't require labeled attack data. It finds what is "normal" and flags what is different — making it powerful for detecting unknown/zero-day threats.
3.4 Network Traffic Analysis

What is Network Traffic Analysis (NTA)?

Network Traffic Analysis (NTA) is the process of monitoring and examining data moving across a network to understand its behavior and performance. It helps security teams detect threats, analyze communication patterns, and maintain a secure and reliable network.

  • Monitors real-time traffic to identify anomalies
  • Detects cyber threats like malware, scans, and data exfiltration
  • Helps troubleshoot performance issues and bottlenecks
  • Provides visibility into user, device, and application behavior
  • Supports investigations with stored logs and packet data

Importance of NTA

Early Threat Detection

Detect anomalies like sudden traffic spikes or odd connection attempts before major damage occurs.

Unmasking Stealthy Attacks

NTA inspects packet details to reveal threats hiding within normal-looking traffic.

Pattern Recognition

Learns recurring behaviors and alerts when suspicious patterns reappear.

Adaptive Defense

Continuous traffic monitoring helps quickly patch new weaknesses as attacker methods evolve.

Unsupervised Learning in NTA

Unsupervised learning algorithms analyze network traffic to identify normal network behavior and detect deviations. The process follows these stages:

Stage 1: Data Gathering and Processing

Network traffic data is collected using different monitoring techniques:

Packet-Level Analysis

Packet capture tools (Wireshark, tcpdump) record every packet including: source/destination IP, protocol type (TCP/UDP/ICMP), packet size, header information, payload data. Data format: PCAP files.

Flow-Level Analysis

Flow-based monitoring (NetFlow, IPFIX) and Firewall/IDS logs. Data formats: CSV logs, NetFlow records.

Stage 2: Feature Extraction

Feature extraction converts raw network packets into meaningful numerical values usable by ML algorithms. Raw packets contain low-level data (headers, payload, binary) which cannot be directly used. Key features extracted include:

  • Flow duration
  • Total packets and total bytes
  • Packet size mean
  • Protocol type
  • TCP flags
  • Inter-arrival time
After feature extraction, raw traffic is converted into a tabular dataset (rows and columns) suitable for unsupervised learning analysis.

Stage 3: Data Preprocessing

Preprocessing is crucial before applying unsupervised learning. Raw data may contain noise, missing values, and inconsistent formats. Main steps:

  • Removing Missing Values
  • Encoding Categorical Features
  • Data Normalization — unsupervised algorithms are sensitive to data scale

Stage 4: Apply Unsupervised Algorithm

K-Means Clustering

Groups traffic into clusters. Points far from clusters = suspicious.

DBSCAN

Better for anomaly detection — marks low-density points as noise/attack traffic.

Autoencoders (Deep Learning)

Learn normal traffic patterns; high reconstruction error = anomaly.

Stage 5: Identify and Validate Anomalies

After model execution, the following are flagged as suspicious: small clusters, noise points, outliers, high reconstruction error. Security team then checks flagged traffic, confirms whether it is an attack or false alarm, and improves feature selection accordingly.

3.5 Monitoring User and Entity Behavior (UEBA)

What is UEBA?

User and Entity Behavior Analytics (UEBA) is a cybersecurity approach that uses smart algorithms and machine learning to spot unusual activities across networks. It detects suspicious activities within networks by analyzing how effectively users operate inside the network — before major consequences occur.

UEBA does not focus on preventing threats from entering, but rather on detecting abnormal behavior of users and entities already inside the network.

How UEBA Works

UEBA constantly collects and appraises vast amounts of information on user behaviors within applications, traffic patterns across networks, and device operations. Using AI, UEBA establishes the characteristic (baseline) behavior of an individual or system. Any variation from this baseline triggers alerts to the security team.

Examples of variations UEBA detects: data accesses outside job description, uncommon login credential attempts, abnormal application usage patterns.

UEBA Use Cases

Insider Threats

Spots when employees or insiders behave abnormally, indicating they may have gained unauthorized entry into systems or resources.

Compromised Accounts

Recognizes strange login activities or attempts to access sensitive data, indicating stolen credentials.

Intrusion Detection

Immediate spikes in data downloads or abnormal traces in network traffic indicate possible breaches or infections.

How UEBA Enables Faster Threat Detection

Understanding Normal Behavior: UEBA establishes what is normal for any individual or entity based on past activities — typical login times, file access patterns, interaction modes
Detecting Unusual Behavior: Intelligent software recognizes unusual present-time activities — logging in from unfamiliar locations or accessing rarely-accessed sensitive data
End-to-End Security: UEBA monitors everyone's patterns simultaneously — capable of detecting coordinated attacks where multiple users behave oddly at once

Detecting Deviations Using Unsupervised Learning

Unsupervised anomaly detection identifies unusual patterns without requiring labeled examples. Useful when anomalies are rare, diverse, or unknown in advance. The key assumption: anomalies are statistically or structurally different from the majority of data.

Common Approaches

MethodTechniqueHow it Detects Anomalies
StatisticalZ-Score / PercentilesPoints falling far from the mean (many standard deviations away) or beyond percentile thresholds are flagged
Distance / Density-BasedKNN, Local Outlier Factor (LOF)Points far from dense clusters of normal data are anomalies; effective for multidimensional datasets
Clustering-BasedK-Means, DBSCANPoints not belonging to any cluster or belonging to very small clusters are anomalies
Machine LearningIsolation ForestPoints isolated quickly (short tree path) are anomalies
Machine LearningOne-Class SVMLearns boundary around normal data; flags points outside this boundary
Machine LearningAutoencodersHigh reconstruction error indicates anomalous input
Time-SeriesMoving Averages, Seasonal DecompositionDeviations from moving average or residual components after decomposition flag anomalies

Applications

  • Cybersecurity and Fraud Detection: Identifying unusual login patterns, transactions, or network activity
  • Structural Health Monitoring (SHM): Detecting changes in bridge vibrations or modal frequencies indicating damage
  • Traffic Flow Analysis: Detecting abnormal traffic conditions or congestion patterns
  • Industrial Monitoring: Flagging equipment failures or sensor malfunctions in manufacturing systems
UEBA vs Traditional Security: Traditional tools (firewalls, signature-based IDS) block known threats at the perimeter. UEBA works inside the network, catching threats that have already bypassed the perimeter by monitoring behavioral anomalies over time.
