Module 1 — Introduction to ML & Cybersecurity

CO1 · 6 Hours

1.1 Types of Machine Learning

What is Machine Learning?

ML is a branch of AI that develops algorithms allowing computers to learn from data and make predictions or decisions without being explicitly programmed.

4 Types

Supervised Learning

Trains on labeled data. Each input has a corresponding output. Model learns input→output mapping to predict on new data.

Unsupervised Learning

Learns patterns from unlabeled data. No outputs used. Finds structure or clusters in data. No feedback from environment.

Semi-Supervised Learning

Between supervised and unsupervised. Uses small labeled + large unlabeled dataset. ML infers labels for unlabeled samples.

Reinforcement Learning

Agent learns by trial and error. Receives rewards/penalties from environment. No sample data — learns from actions.

ML Pipeline Stages

Data Collection → Data Pre-Processing → Choosing the Right Model → Training the Model → Evaluating the Model → Hyperparameter Tuning → Predictions & Deployment

1.2 Model Families, Loss Functions, Optimization

Model Families

Every ML algorithm is defined by three components: Model Family · Loss Function · Optimization Procedure.

A model family defines the set of possible models described by a finite set of parameters. Learning = selecting parameter values that best fit training data.

The simplest model is a linear decision boundary (hyperplane): H: θ·x − β = 0. To assign probabilities, the sigmoid (logistic) function maps real-valued scores to [0,1].

Loss Functions

A loss function quantitatively measures error between predicted and actual outputs. The optimal model minimizes loss over training data.

Regression — Sum of Squared Errors

Loss = Σ(yᵢ − ŷᵢ)². Minimized using gradient descent iteratively.

Classification — Negative Log Likelihood

For logistic regression. Penalizes incorrect and overconfident predictions more strongly. Differentiable → suitable for gradient optimization.

Optimization

Goal: find parameters θ that minimize the loss function.

TypeAlgorithmKey Points
First-orderGradient DescentUses 1st derivative; iteratively moves params in direction that reduces loss
First-orderSGDStochastic — better at escaping local minima; high variance oscillations; use decreasing learning rate
First-order (adaptive)Adam, AdaGrad, AdaDeltaSeparate adaptive learning rates per parameter; solve SGD problems
Second-orderNewton-basedUses 2nd derivative; no slow convergence paths; good at saddle points; slower to compute
No single optimizer is universally optimal. Choice depends on: dataset size · nature of loss function · classification vs regression · speed/resource requirements.
1.3 Proactive & Reactive Security
Proactive Security

Designed to maintain overall security even if individual components are compromised. Prevents threats before they occur.

PPDM (Privacy-Preserving Data Mining) — protects private data during mining. Methods: data distribution, modification, rule hiding, privacy-preservation.

Reactive Security

Responds to threats after an attack. Focuses on identifying known attack patterns and taking corrective actions. Primary objective: damage control and recovery.

Key component: IDS — monitors activities, analyzes events, generates reactions to intrusions.

IDS Detection Modules

1. Misuse / Signature Detection

Triggers alarm when a known attack pattern is detected. Fast for known attacks. Cannot detect novel/zero-day attacks.

2. Anomaly Detection

Detects deviations from a learned "normal" profile. Can detect unknown attacks. High false alarm rate.

3. Hybrid Detection

Combines misuse + anomaly to compensate for individual weaknesses. More effective overall.

4. Scan Detection

Generates alerts when attackers scan services before launching attacks. Identifies precursors of attacks.

5. Profiling Modules

Groups similar network connections using clustering algorithms to find dominant (normal) behaviors.

1.4 Real-World Uses of ML in Security

Two Broad ML Use Cases in Security

Pattern Recognition

Discovers explicit/latent characteristics in data. Feature sets teach an algorithm to recognize forms of data with same characteristics.

Examples: Spam filtering, Malware detection, Botnet detection

Anomaly Detection

Establishes a notion of normality (95%+ of data). Any deviation is flagged as anomalous. Infinite anomalous patterns possible — even from hypothetical data.

Examples: Network outlier detection, Malicious URL detection

Key Security Applications

ApplicationML Role
Intrusion Detection (IDS)Builds classifier distinguishing normal vs malicious connections. Learns from labeled network data.
Access ControlUnsupervised learning infers normal access patterns; anomaly detection triggers on deviations.
Spam DetectionPattern recognition on spam characteristics (keywords, links, sender). Also frameable as anomaly detection.
Malware/Botnet DetectionPattern recognition; handles polymorphic malware. Static + dynamic feature analysis.
FuzzingML learns patterns of previously found vulnerabilities to guide fuzzers to vulnerable code paths.
Network Outlier DetectionMost network traffic follows strict protocols — deviations caught by outlier algorithms.

Module 2 — Supervised Learning for Cybersecurity

CO2 · 12 Hours

2.1 Overview of Supervised Learning

How It Works

Labeled dataset → model identifies patterns between input features and outputs → minimizes errors using a loss function → generalizes to predict on new data.

Classification

Output is a category. E.g., "spam" or "not spam". Predicts discrete labels.

Regression

Output is a continuous value. E.g., house price prediction. Predicts numerical quantities.

Importance in Cybersecurity

  • Malware Detection: Classifies malware from file attributes/behavior. Random Forests + Gradient Boosting improve accuracy.
  • Real-Time Threat Detection: Analyzes network traffic and system calls to predict malicious activity.
  • Continuous Improvement: Models retrain with updated data to adapt to evolving threats.
2.2 Data Collection, Preprocessing, Labeling & Feature Extraction

Sources of Cybersecurity Data

Network Traffic Logs

IP addresses, protocols, ports, packet sizes, timestamps. Supports anomaly detection.

System Logs

OS/application events: logins, file access, config changes, failed logins, suspicious activity.

Endpoint Data

Laptops, phones, IoT devices. Detects compromised devices and malware.

Threat Intelligence Feeds

Known vulnerabilities, malicious IPs and domains. Neutralizes known actors.

User Behaviour Data

Logins, file access, application usage. Reveals insider threats and compromised accounts.

Data Preprocessing Pipeline

Collection → Cleaning (remove irrelevant data, handle missing values, normalize) → Feature Extraction → Model Training → Evaluation (accuracy, precision, recall, F1)

Data Labeling Techniques

Assigning meaningful tags/categories to raw data so ML models can distinguish normal vs malicious behavior.

TechniqueHowPros / Cons
Manual (Expert-Based)Cybersecurity experts inspect and label data✅ High accuracy — ❌ Slow, expensive, not scalable
Signature-BasedPredefined attack signatures auto-label data✅ Efficient for known attacks — ❌ Misses zero-day
Rule-BasedThresholds/heuristics (e.g., repeated failed logins = brute force)✅ Simple — ❌ False positives if rules outdated
Log & Alert-BasedIDS/IPS/SIEM alerts directly used as labels✅ Large-scale automated — ❌ Inherits tool biases
Honeypot-BasedAny interaction with decoy system = malicious label✅ High-confidence malicious labels — ❌ No normal traffic
Semi-SupervisedSmall labeled + large unlabeled; model infers rest✅ Reduces manual effort — ❌ Errors propagate
Active LearningModel requests expert to label only uncertain samples✅ Minimizes cost — ❌ Continuous expert involvement

Feature Extraction

Converts raw data into relevant structured features for ML models.

From Network Traffic
  • Packet-based: size, count, protocol
  • Flow-based: duration, bytes sent/received
  • Time-based: connection rate, inter-arrival time
  • Statistical: mean, variance, entropy
From Malware Files
  • Static: file size, header info, opcode sequences, API calls
  • Dynamic: system calls, network behavior, file modifications
From System Logs
  • Login attempt count, system call frequency
  • Access time patterns, error/warning counts
From Text (Spam/Phishing)
  • BoW: word frequency vector
  • TF-IDF: weights rare words higher
  • N-grams: opcode/word sequences

Feature Selection

Choosing most relevant features. Reduces dimensionality, improves accuracy, reduces overfitting.

MethodHowExamples
FilterStatistical properties; no ML model needed. Fast and scalable.Information Gain, Chi-square, Correlation, Mutual Information
WrapperUses ML model to evaluate feature subsets. High accuracy but expensive.Forward Selection, Backward Elimination, RFE
EmbeddedFeature selection during model training. Balances efficiency + accuracy.Decision Trees, Random Forest importance, LASSO (L1)
Overfitting: model too complex, memorizes noise — poor on test data. Underfitting: model too simple, high bias — poor on both train and test.
2.3 Intrusion Detection Systems (IDS)

What is IDS?

Software that monitors network/system activities for malicious activity or policy violations. Builds a classifier distinguishing normal vs attack connections. Alerts logged via SIEM or sent to admin.

Types of IDS

TypeDescription
NIDS (Network)Monitors entire subnet traffic; matches against known attack signatures
HIDS (Host)Runs on individual device; monitors incoming/outgoing packets; snapshots system files for comparison
Signature-BasedCompares packets to attack signature database; requires regular updates; can't detect unknown attacks
HybridCombines host agent + network info; most effective
PIDS (Protocol-Based)Resides at front of server; monitors HTTPS/HTTP protocol stream
APIDS (App Protocol-Based)Resides within server group; monitors application-specific protocol communication

Common Intrusion Methods

  • Address Spoofing: fake source IP using proxy servers
  • Fragmentation: sending data in small pieces to slip past detection
  • Pattern Evasion: changing attack methods to avoid signature matching
  • Coordinated Attack: multiple attackers/ports to confuse IDS

Supervised Learning Techniques for IDS

Decision Trees

Binary tree structure; splits dataset top-down using conditions. Analyzes packet size, protocol type, connection duration, traffic rate to classify normal vs intrusive activity.

  • Split metrics: Gini Impurity (lower = purer nodes), Information Gain (higher = better split), Variance Reduction
  • Stop splitting when: all leaves pure OR max depth reached OR child node too small
  • ✅ Easy to interpret, fast, supports numerical + categorical features
  • ❌ Prone to overfitting; use pruning to reduce complexity. Small data changes → large tree changes.

Random Forests

Ensemble of multiple decision trees. Combines outputs to improve accuracy and reduce overfitting. Handles noisy/imbalanced data well. Provides feature importance ranking.

Support Vector Machines (SVM)

Linear classifier producing a maximum-margin hyperplane separating two classes. Uses hinge loss (penalizes points on wrong side). Kernel trick enables non-linear boundaries.

  • ✅ Handles high-dimensional data; effective for malware detection and IDS

Neural Networks

Layers of connected nodes mimicking brain neurons. CNNs/RNNs/MLPs analyze network traffic for anomalies. Can achieve >99% accuracy.

  • ✅ Automatic feature extraction, real-time monitoring, adaptable to new threats, reduces false positives
Variance Reduction Techniques: Cross-Validation · Regularization (L1/L2) · Pruning · Feature Selection
2.4 DDoS Attack Detection

What is DDoS?

Distributed Denial-of-Service: floods target with traffic from multiple compromised devices (botnets/zombies). Results in degraded performance or service outage. Attacker sends remote instructions to each bot.

Traffic Analysis — Signs of DDoS

  • Suspicious traffic volume from a single IP or IP range
  • Flood from users sharing same device type/geolocation/browser
  • Unexplained surge in requests to single endpoint
  • Unnatural traffic patterns (e.g., spikes every 10 minutes)

Key Features Extracted for DDoS Detection

FeatureSignificance
Packet RateAbnormally high rate = flooding attack indicator
Request RateSharp increase = application-layer (HTTP flood) attack
Source IP CountHigh unique IPs = distributed botnet attack
Packet SizeUniform sizes = protocol abuse/flooding pattern
Flow DurationVery short flows = repeated connections; very long = resource exhaustion
Protocol TypeDominance of one protocol (UDP/ICMP floods) = protocol-specific attack

Supervised Learning Approach

Label network traffic as normal or DDoS attack → train model → classify incoming traffic in real time → activate mitigation when attack detected.

Algorithms used: Decision Trees, Naive Bayes, Logistic Regression, SVM, Random Forests

Evaluation: Accuracy, Precision, Recall, False Positive Rate. High recall is critical — must not misclassify attack as normal.

✅ Advantages
  • High detection accuracy
  • Learns complex non-linear patterns
  • Automated and adaptive
  • Suitable for real-time monitoring
❌ Limitations
  • Requires large labeled datasets
  • Depends on data quality and feature selection
  • Limited against previously unseen attack patterns
2.5 Spam Filtering — Naive Bayes & Logistic Regression

Characteristics of Spam Emails

  • Promotional/misleading content with exaggerated offers and urgency
  • Frequent spam keywords: free, win, urgent, click here
  • Suspicious links, shortened URLs, unexpected attachments
  • Spoofed or unknown sender info; mismatch between name and address
  • Poor grammar, no personalization, generic greetings ("Dear User")
  • HTML tricks: hidden text, large fonts, bright colors
  • High sending frequency from automated systems

NLP Techniques for Email Processing

Text Mining + NLP pipeline: Information Retrieval → Text Cleaning (lowercase, remove punctuation) → Tokenization → Stop-Word Removal → Stemming/Lemmatization → Feature Representation

Bag of Words (BoW)

Represents word frequency. Simple but ignores word order and context.

TF-IDF

Term Frequency × Inverse Document Frequency. Weights rare/important words higher; down-weights common words.

N-grams

Sequences of N words/characters. Captures local context and word patterns.

Sentiment Analysis

Identifies emotional tone (positive/negative/neutral). Used as additional feature to improve classification accuracy.

Naive Bayes Classifier

Naive Bayes predicts category using probability. "Naive" = assumes all features are independent (never actually true, but works well in practice for spam).

Core idea: Given features X = {x₁,…,xₙ}, compute P(Y=C | X) using Bayes' Theorem. Choose class C with highest posterior probability.

  • Prior P(C): proportion of class C in training data
  • Likelihood P(xᵢ|C): probability of word given class
  • Laplace Smoothing: adds "phantom" samples to handle unseen words (avoids zero probabilities)
  • Log-Probability: use log to avoid numerical underflow from multiplying tiny probabilities
In practice, compute the ratio of probability estimates for class C1 vs C2. Score > 1 → C1 more likely. Score < 1 → C2 more likely.

Logistic Regression

Takes numerical feature vectors, predicts log odds of each class. Sigmoid function maps output to probability [0,1].

Decision boundary is linear in log-odds space. Threshold (e.g., 0.5): above → spam, below → not spam.

✅ Advantages
  • Trains efficiently, scales to millions of features
  • Explainable — each feature's contribution computable
  • Fast scoring (dot product)
❌ Limitations
  • Assumes linearity between features and log odds
  • Features must be independent
  • Requires larger sample size than some other algorithms

Implementation Pipeline (Logistic Regression for Spam)

Load CSV data → Train/Test split (e.g., 80/20) → TF-IDF vectorization → Fit LogisticRegression model → Predict on test set → Evaluate with accuracy_score

Evaluation Metrics: Accuracy = correct predictions / total. Precision = true spam / classified spam. Recall = true spam / all actual spam. F1 = harmonic mean of precision and recall.
